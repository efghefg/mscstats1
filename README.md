# Редактор отчёта по СИМ — версия 2

Статический HTML/CSS/JavaScript-проект без npm, Vite и внешних библиотек. Его можно публиковать на GitHub Pages напрямую.

## Что исправлено

- Все изменяемые показатели и графики рисуются JavaScript на Canvas 1920×1080.
- Превью и PNG создаются одним и тем же рендерером, поэтому при скачивании ничего не смещается.
- Пустые страницы из PDF используются как точная статическая подложка. Подложка всегда рисуется внутрь Canvas и сохраняется в PNG.
- Стрелки и цвета рассчитываются автоматически:
  - рост поездок — зелёная стрелка вверх;
  - снижение поездок — красная стрелка вниз;
  - рост ДТП — красная стрелка вверх;
  - снижение ДТП — зелёная стрелка вниз.
- Доли Mos ID, доли ДТП, итоги квот и таблицы нарушений считаются автоматически.
- Столбчатые диаграммы масштабируются по введённым значениям и имеют фиксированную геометрию.
- Подключение Moscow Sans уже настроено для файла `assets/moscow-sans-regular.ttf`. Перед публикацией положите в эту папку ваш локальный файл шрифта.
- Supabase работает через REST API, поэтому ошибки npm-импортов на GitHub Pages больше нет.

## Публикация на GitHub Pages

1. Удалите старое содержимое репозитория.
2. Загрузите **содержимое** этой папки в корень репозитория. `index.html` должен лежать в корне.
3. Сделайте commit и push в ветку `main`.
4. Откройте `Settings → Pages` и выберите `Source: GitHub Actions`.
5. Дождитесь зелёной галочки во вкладке `Actions`.

Проект не требует сборки. Все пути относительные и работают в подпапке репозитория.

## Supabase

1. Откройте SQL Editor в Supabase.
2. Выполните файл `supabase.sql`.
3. В редакторе раскройте блок «Подключение Supabase».
4. Укажите Project URL, anon key и таблицу `sim_report_slides`.
5. Нажмите «Проверить».

Кнопка «Сохранить слайд» выполняет upsert текущего слайда. Кнопка «Обновить данные» загружает текущий слайд из Supabase. Без Supabase данные сохраняются в `localStorage` браузера.

## Файлы

- `index.html` — интерфейс;
- `styles.css` — адаптивный дизайн редактора;
- `app.js` — формы, расчёты, Canvas-отрисовка, экспорт и Supabase;
- `assets/slide-*-bg.png` — пустые страницы из приложенного PDF;
- `assets/moscow-sans-regular.ttf` — сюда нужно добавить ваш локальный файл Moscow Sans;
- `supabase.sql` — таблица и политики;
- `.github/workflows/deploy.yml` — публикация GitHub Pages.
